      <h1 class="post-title entry-title">MySQL 教程</h1>
      <hr>
<p class="post-byline"><span class="vcard author">
    
             
               </p>
      
      <div class="clear"></div>

      <div class="entry themeform share">
        <div class="entry-inner">
	<p>
	<img src="/study/uploads/201812/7816267010a83b0a5b3cd3e24083ce82.jpg" alt="" />

MySQL是一個關係型資料庫管理系統，由瑞典MySQL AB 公司開發，目前屬於Oracle旗下產品。MySQL 是最流行的關係型資料庫管理系統之一，在 WEB 應用方面，MySQL是最好的 RDBMS (Relational Database Management System，關係資料庫管理系統) 應用軟體。
<p>MySQL是一種關係資料庫管理系統，關係資料庫將資料儲存在不同的表中，而不是將所有資料放在一個大倉庫內，這樣就增加了速度並提高了靈活性。
<p>MySQL所使用的 SQL 語言是用於訪問資料庫的最常用標準化語言。MySQL 軟體採用了雙授權政策，分為社群版和商業版，由於其體積小、速度快、總體擁有成本低，尤其是開放原始碼這一特點，一般中小型網站的開發都選擇 MySQL 作為網站資料庫。
<p>由於其社群版的效能卓越，搭配 PHP 和 Apache 可組成良好的開發環境。
</p>
<p>
	在本教程中，會讓大家快速掌握 MySQL 的基本知識，並輕鬆使用 MySQL 資料庫。
</p>
<hr />
<h2>
	什麼是資料庫？
</h2>
<p>
	資料庫（Database）是按照資料結構來組織、儲存和管理資料的倉庫，
</p>
<p>
	每個資料庫都有一個或多個不同的API用於建立，訪問，管理，搜尋和複製所儲存的資料。
</p>
<p>
	我們也可以將資料儲存在檔案中，但是在檔案中讀寫資料速度相對較慢。
</p>
<p>
	所以，現在我們使用關係型資料庫管理系統（RDBMS）來儲存和管理的大資料量。所謂的關係型資料庫，是建立在關係模型基礎上的資料庫，藉助於集合代數等數學概念和方法來處理資料庫中的資料。
</p>
<p>
	RDBMS即關係資料庫管理系統(Relational Database Management System)的特點：
</p>
<ul>
	<li>
		1.資料以表格的形式出現
	</li>
	<li>
		2.每行為各種記錄名稱
	</li>
	<li>
		3.每列為記錄名稱所對應的資料域
	</li>
	<li>
		4.許多的行和列組成一張表單
	</li>
	<li>
		5.若干的表單組成database
	</li>
</ul>
<hr />
<h2>
	RDBMS 術語
</h2>
<p>
	<br />
</p>
<p>
	在我們開始學習MySQL 資料庫前，讓我們先了解下RDBMS的一些術語：
</p>
<ul>
	<li>
		<strong>資料庫:</strong> 資料庫是一些關聯表的集合。
	</li>
	<li>
		<strong>資料表:</strong> 表是資料的矩陣。在一個數據庫中的表看起來像一個簡單的電子表格。
	</li>
	<li>
		<strong>列:</strong> 一列(資料元素) 包含了相同的資料, 例如郵政編碼的資料。
	</li>
	<li>
		<strong>行：</strong>一行（=元組，或記錄）是一組相關的資料，例如一條使用者訂閱的資料。
	</li>
	<li>
		<strong>冗餘</strong>：儲存兩倍資料，冗餘降低了效能，但提高了資料的安全性。
	</li>
	<li>
		<strong>主鍵</strong>：主鍵是唯一的。一個數據表中只能包含一個主鍵。你可以使用主鍵來查詢資料。
	</li>
	<li>
		<strong>外來鍵：</strong>外來鍵用於關聯兩個表。
	</li>
	<li>
		<strong>複合鍵</strong>：複合鍵（組合鍵）將多個列作為一個索引鍵，一般用於複合索引。
	</li>
	<li>
		<strong>索引：</strong>使用索引可快速訪問資料庫表中的特定資訊。索引是對資料庫表中一列或多列的值進行排序的一種結構。類似於書籍的目錄。
	</li>
	<li>
		<strong>參照完整性:</strong> 參照的完整性要求關係中不允許引用不存在的實體。與實體完整性是關係模型必須滿足的完整性約束條件，目的是保證資料的一致性。
	</li>
</ul>
<p>
	MySQL 為關係型資料庫(Relational Database Management System), 這種所謂的"關係型"可以理解為"表格"的概念, 一個關係型資料庫由一個或數個表格組成, 如圖所示的一個表格:
</p>
<p>
	<img src="/study/uploads/201812/2124d0038dae56b51d22f2853353b3c5.jpg" alt="" /> 
</p>
<ul>
	<li>
		表頭(header): 每一列的名稱;
	</li>
	<li>
		列(col): 具有相同資料型別的資料的集合;
	</li>
	<li>
		行(row): 每一行用來描述某條記錄的具體資訊;
	</li>
	<li>
		值(value): 行的具體資訊, 每個值必須與該列的資料型別相同;
	</li>
	<li>
		<strong>鍵(key)</strong>: 鍵的值在當前列中具有唯一性。
	</li>
</ul>
<hr />
<h2>
	MySQL資料庫
</h2>
<p>
	MySQL 是一個關係型資料庫管理系統，由瑞典 MySQL AB 公司開發，目前屬於 Oracle 公司。MySQL 是一種關聯資料庫管理系統，關聯資料庫將資料儲存在不同的表中，而不是將所有資料放在一個大倉庫內，這樣就增加了速度並提高了靈活性。
</p>
<ul>
	<li>
		MySQL 是開源的，所以你不需要支付額外的費用。
	</li>
	<li>
		MySQL 支援大型的資料庫。可以處理擁有上千萬條記錄的大型資料庫。
	</li>
	<li>
		MySQL 使用標準的SQL資料語言形式。
	</li>
	<li>
		MySQL 可以運行於多個系統上，並且支援多種語言。這些程式語言包括C、C++、Python、Java、Perl、PHP、Eiffel、Ruby和Tcl等。
	</li>
	<li>
		MySQL 對PHP有很好的支援，PHP是目前最流行的Web開發語言。
	</li>
	<li>
		MySQL 支援大型資料庫，支援5000萬條記錄的資料倉庫，32位系統表文件最大可支援4GB，64位系統支援最大的表文件為8TB。
	</li>
	<li>
		MySQL 是可以定製的，採用了GPL協議，你可以修改原始碼來開發自己的 MySQL 系統。
	</li>
</ul>
